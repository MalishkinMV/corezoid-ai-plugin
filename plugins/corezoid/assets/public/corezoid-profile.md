# Corezoid Public Product Profile

This file captures the public product context used to prepare the Codex plugin card and guidance. It is intentionally concise so the plugin stays product-oriented while the full technical corpus remains bundled under `../source/`.

## Positioning

Corezoid is presented publicly as an Actor/Process Engine for building and running business processes, APIs, integrations, dashboards, and automation flows. The plugin should therefore be framed as a universal Corezoid product assistant, not only as a wrapper around the `corezoid-ai-doc` repository.

## Product Areas To Surface

- Process design and execution: drag-and-drop process editing, real-time view mode, debugging, one-click deployment, and execution without rebooting the whole system.
- Organization and access: projects, components, processes, users, groups, API keys, permissions, task management, collaboration, and transparency.
- Analytics: real-time dashboards, customizable dashboards, activity monitoring, task/state/traffic analysis, and operational visibility.
- API management: external API orchestration, unified API management, automatic API creation from processes, webhooks, performance, and access-rights security.
- Learning and enablement: quick start, tutorials, documentation, workshops, certification, and FAQ.
- Developer references: public GitHub organization, `corezoid-ai-doc`, schemas, templates, playbooks, samples, and MCP server reference material.

## Codex Plugin Promise

Use this plugin when a user wants to:

- Design a Corezoid architecture before building.
- Build connector or logic process JSON.
- Wrap external APIs or expose Corezoid processes as APIs.
- Review process JSON before deployment.
- Modify/debug existing processes and failed tasks.
- Generate process documentation and enrich process JSON.
- Plan real-time dashboards and operational monitoring.
- Navigate Corezoid docs, schemas, templates, and public reference material.
